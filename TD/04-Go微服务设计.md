# Go 数据服务设计(文件4)

> 深化 [`01-总览.md`](01-总览.md) 第 4 节。对应实现目录 `service-data/`。
> 职责:词典抓取 + MySQL 持久化 + SM-2 复习调度,对 Python 侧暴露 HTTP/JSON REST。**不含任何 LLM/对话逻辑。**

## 1. 分层与目录

```
service-data/
├── cmd/server/main.go        # 装配:读 env → 连 MySQL → 起 HTTP
├── internal/
│   ├── model/model.go        # DTO(既是 JSON 出参也是内部结构)
│   ├── dict/dict.go          # DictionaryAPI.dev 客户端 + 解析
│   ├── sm2/sm2.go            # SM-2 纯算法(+ sm2_test.go)
│   ├── store/store.go        # MySQL 访问 + 事务
│   └── api/api.go            # HTTP 路由 + handler + 编解码
└── migrations/schema.sql     # 建表
```

依赖方向单向:`api → store → (sm2, model)`,`api → dict`,`store → (sm2, model)`。
`sm2` 无任何 I/O 依赖,可独立单测。

**唯一第三方依赖**:`github.com/go-sql-driver/mysql`(MySQL 驱动)。路由用标准库 `net/http`(Go 1.22+ 的方法+通配路由),JSON 用 `encoding/json`。

## 2. 配置(环境变量)

| 变量 | 默认 | 说明 |
|---|---|---|
| `MYSQL_DSN` | `root@tcp(127.0.0.1:3306)/wordbot?parseTime=false&charset=utf8mb4` | go-sql-driver DSN |
| `HTTP_ADDR` | `:8080` | 监听地址 |
| `DICT_API_BASE` | `https://api.dictionaryapi.dev/api/v2/entries/en` | 词典 API 前缀 |

> `parseTime=false`:`DATETIME` 以文本返回,直接 `Scan` 进 `string`(`due_at` 等原样透传给前端)。
> 连接池:`SetMaxOpenConns(10)`、`SetConnMaxLifetime(3m)`。启动时 `db.Ping()`,失败即 `Fatal`。

## 3. REST 契约

**通用约定**
- 请求/响应体均 JSON;成功 `200`。
- 错误统一形态:`{"error": "<message>"}`,配合 HTTP 状态码。
- `tg_user_id`、`word_id`、`user_word_id` 均为整数,随 query 或 body 传入。

### 3.1 `GET /healthz`
存活检查。→ `200 {"status":"ok"}`。不依赖 DB 逻辑(但进程能起来即表示启动 Ping 已过)。

### 3.2 `GET /words/{text}` — 查词(核心)
路径参数 `text` 会被小写+trim 归一化。

流程:命中 `words` 缓存 → 直接返回;否则调 DictionaryAPI → 解析 → 事务落库 → 返回。

**200** 响应(`model.Word`):
```json
{
  "id": 12,
  "text": "resilient",
  "phonetic": "/rɪˈzɪljənt/",
  "audio_url": "https://.../resilient-us.mp3",
  "senses": [
    {"pos":"adjective","meaning_en":"...","examples":["..."],
     "synonyms":["tough"],"antonyms":["fragile"]}
  ]
}
```

| 状态码 | 场景 |
|---|---|
| `400` | text 为空 |
| `404` | 词典未收录(`dict.ErrNotFound`) |
| `503` | 词典不可用(超时/网络/非 200) |
| `500` | DB 错误 |

### 3.3 `POST /notebook` — 加入单词本(幂等)
Body:`{"tg_user_id":123,"word_id":12}`
- 幂等:重复收藏返回同一行(`ON DUPLICATE KEY UPDATE`)。
- 新卡:`ease_factor=2.50, interval_days=0, repetitions=0, due_at=NOW(), status=0`。

**200**(`model.UserWord`):
```json
{"id":5,"tg_user_id":123,"word_id":12,"ease_factor":2.5,
 "interval_days":0,"repetitions":0,"due_at":"2026-07-04 10:00:00","status":0}
```
错误:`400`(缺字段)、`500`。

### 3.4 `GET /reviews/due?tg_user_id=&limit=` — 今日到期
- `tg_user_id` 必填;`limit` 可选(默认 20)。
- 取 `due_at<=NOW()` 的卡片,按 `due_at` 升序。

**200**:`{"cards":[ {DueCard...} ]}`,`DueCard` 含 `user_word_id/word_id/text/phonetic/senses`。
错误:`400`(缺 tg_user_id)、`500`。

### 3.5 `POST /reviews` — 提交复习(SM-2,事务)
Body:`{"user_word_id":5,"quality":3}`,`quality∈[0,5]`(越界会被 `sm2` 钳制)。

**200**:更新后的 `model.UserWord`(`interval_days` 即「下次几天后」)。
错误:`400`(缺 user_word_id)、`500`(卡片不存在 / 事务失败)。

### 3.6 `GET /words/recent?tg_user_id=&n=` — 近期词
供 Python 侧作为阅读选材。`n` 默认 10。
**200**:`{"words":[ {"id","text","phonetic"} ]}`。

> 路由注册顺序上 `/words/recent` 与 `/words/{text}` 并存;Go 1.22 ServeMux **精确段优先于通配段**,故 `recent` 不会被 `{text}` 吞掉。

### 3.7 `POST /readings` — 阅读留档
Body:`{"tg_user_id":123,"content":"...","target_words":["a","b"],"model":"gemini-2.5-flash"}`
**200**:`model.Reading`(含新 `id`)。错误:`400`(缺 tg_user_id/content)、`500`。

### 3.8 `GET /stats?tg_user_id=` — 统计
**200**:`{"total_words","due_today","mastered","reviews_total"}`。

## 4. 词典抓取(dict 包)

- `Fetch(ctx, text)`:`GET {base}/{urlEscape(text)}`,`http.Client` 超时 **5s**。
- 响应是 entry 数组;映射:`meanings[].definitions[]` → 每个 definition 一个 `Sense`(带该 meaning 的 `partOfSpeech`)。
- 同/反义词:优先取 definition 级,回退 meaning 级(`pick`)。
- 音标:`phonetic` 空则取首个非空 `phonetics[].text`;音频取首个非空 `phonetics[].audio`。
- 义项上限 `maxSenses=6`。
- `404` → `ErrNotFound`;空数组/无义项 → `ErrNotFound`;其它非 200 → error。

## 5. SM-2 调度(sm2 + store)

**纯算法**(`sm2.Next(state, q)`,见 03 §3.3 / 01 §3.7):
- `q<3`:`repetitions=0`,`interval=1`(明天重来)。
- `q>=3`:`interval` 依 `repetitions` 取 1 → 6 → `round(interval*EF)`;`repetitions++`。
- `EF += 0.1-(5-q)*(0.08+(5-q)*0.02)`,下限 `1.30`。
- `q` 越界钳到 `[0,5]`。

**事务落库**(`store.SubmitReview`):
```
BEGIN
  SELECT ... FROM user_words WHERE id=? FOR UPDATE   -- 行锁,防并发
  next = sm2.Next(cur, quality)
  status = 2 if next.Interval >= 21 else 1
  UPDATE user_words SET ef,interval,reps,due_at=DATE_ADD(NOW(),INTERVAL next.Interval DAY),
                        last_review_at=NOW(), status
  INSERT review_logs (prev/next interval, prev/next ef, quality)
COMMIT
```
`FOR UPDATE` 保证同卡并发提交串行化。

## 6. 错误处理约定

| HTTP | 含义 | 触发点 |
|---|---|---|
| 400 | 参数缺失/非法 | handler 校验、JSON 解码失败 |
| 404 | 资源不存在 | 词典未收录 |
| 503 | 下游不可用 | 词典 API 超时/网络 |
| 500 | 内部错误 | DB/事务失败 |

所有非 2xx 都返回 `{"error": "..."}`;Python 侧 `DataClient` 将 404 映射为 `WordNotFound`,其余 4xx/5xx 映射为 `DataAPIError`。
